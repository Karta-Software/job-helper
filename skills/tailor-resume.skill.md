# tailor-resume

Use when creating a role-specific resume version.

## Steps

1. Read target posting or role criteria.
2. Confirm the posting link is live or intentionally proceed from a saved posting.
3. Build or load the role's skill inventory.
4. Run `experience-finder`.
5. Draft resume JSON or Markdown.
6. Apply `ResumeConstraints`.
7. Run `evidence-auditor`.
8. Run `voice-auditor`.
9. Remove fourth-wall guidance from applicant-facing files.
10. Save a new private resume version.
11. Generate requested formats.
12. Verify page count or render output when required.

## Applicant-Facing Constraints

- Do not include "best target fit," "use this version," first-pass target lists, claim policies, ranking notes, or internal caveats.
- Keep targeting guidance in a separate index, tracker note, or application plan.
- Keep claims evidence-backed.
- Frame adjacent experience honestly.
- Remove do-not-claim skills.
- Respect max page count.
- Prefer ATS-safe formatting unless the target role calls for a designed resume.
