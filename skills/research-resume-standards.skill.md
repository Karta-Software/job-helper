# research-resume-standards

Use when deciding resume length, sections, formatting, content density, or recruiter-conversion rules for a target market.

## Agent Orchestra

1. `researcher` finds current resume, recruiter, ATS, and application-conversion sources.
2. `job-finder` collects representative postings for the target role family.
3. `posting-scorer` extracts repeated hard filters, skills, titles, and seniority signals.
4. `company-researcher` checks whether target companies have special resume or application norms.
5. `network-finder` identifies referral paths that may change how much context the resume needs.
6. `application-strategist` converts findings into pipeline rules.
7. `experience-finder` maps the rules to graph-backed evidence.
8. `resume-writer` proposes section and bullet constraints.
9. `evidence-auditor` grades each rule as source-backed, inferred, or weak.
10. `voice-auditor` catches bloated, AI-sounding, or keyword-stuffed guidance.
11. `devils-advocate` challenges the conclusions like a skeptical recruiter.
12. `decision-helper` turns remaining ambiguity into user choices.
13. `drill-author`, `drill-adversary`, and `drill-realism-auditor` convert resume gaps into interview-prep drills when needed.

## Outputs

- `resume-standards.json`
- source-backed research note in the private graph
- resume scorecard rules for `tailor-resume`
- open questions for the user

## Rules

- Prefer current data over timeless advice.
- Separate evidence from inference.
- Report interview or callback conversion data when available.
- Do not optimize for ATS folklore when recruiter-conversion data conflicts with it.
- Keep page, word, character, line, and bullet targets explicit.
