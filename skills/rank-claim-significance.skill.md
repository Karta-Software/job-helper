# rank-claim-significance

Use when choosing which true skills, experiences, anecdotes, tools, domains, or proof points deserve resume space.

## Rating Concept

This is not a truth audit. Truth is handled by skill ratings and evidence review.

Significance answers a different question: how much should this item shape the candidate's public positioning?

## Tiers

- Headliner: should shape most technical resumes.
- Strong support: often worth a bullet or skills-line slot.
- Role-specific: include when the posting asks for it.
- Interview only: useful story, but not default resume space.
- De-emphasize: true or adjacent, but usually not strategic.

## Steps

1. Load the candidate's confirmed skills, partial skills, major proof points, target roles, and recent resume drafts.
2. Remove do-not-claim items unless the goal is to rank guardrails.
3. Present items in an interactive ranker or a compact grouped list.
4. Preserve order within each tier.
5. Capture optional notes explaining why an item matters or when it should be used.
6. Save the result to the private career graph or private workspace.
7. During resume tailoring, use significance after target-role relevance and claim confidence to decide what gets space.

## Rules

- Do not let significance override truth. Unsupported claims still stay out.
- Do not let one memorable anecdote outrank stronger durable experience unless the target role makes that proof point unusually valuable.
- Treat AI-native development, leadership, domain expertise, and quantified outcomes as potential headliners when the user ranks them that way.
- Re-run the ranking when the target role changes materially.
