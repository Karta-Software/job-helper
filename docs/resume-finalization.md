# Resume Finalization

Use this checklist before a resume is treated as application-ready.

## Applicant-Facing Content

- The resume contains only candidate-facing content.
- Targeting notes, ranking notes, and "use this version" guidance are outside the resume.
- Every claim is verified or explicitly allowed as partial by the resume constraints.
- Adjacent skills are framed honestly.
- Do-not-claim skills are absent.
- Keywords are present only where the candidate can defend them.

## Structure

Recommended one-page sections:

1. Name and contact line.
2. Role-specific headline.
3. Professional summary.
4. Core skills.
5. Experience.
6. Education or credentials.

For most applications, keep one primary experience section and select only the strongest bullets for the role. Put extra targeting notes in the resume index or application plan.

## Formatting Constraints

- Prefer ATS-safe text over complex tables or columns.
- Use consistent heading names.
- Keep bullets outcome-oriented and short enough to scan.
- Avoid icons, text boxes, decorative sidebars, and hard-to-parse layouts unless the target application explicitly benefits from a designed resume.
- Verify the PDF page count when a one-page resume is required.
- If DOCX visual rendering is not available, still perform structural checks and disclose the fallback.

## Evidence Audit

Before finalizing:

- Check each bullet against the career graph.
- Mark unsupported claims for deletion or clarification.
- Keep skill claims aligned with the candidate's direct, adjacent, and do-not-claim ratings.
- Avoid upgrading adjacent experience into direct ownership.
- Avoid adding industry expertise just because the posting asks for it.

## Handoff Files

For each final resume version, keep:

- editable Markdown or JSON source
- DOCX version for application systems that request Word
- PDF version for review and PDF uploads
- separate index/README with targeting guidance
- claim-evidence map when available
